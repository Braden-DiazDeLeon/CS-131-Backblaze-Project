jobs/failure_rate_spark.py

We use an explicit schema for the first five columns instead of relying on `inferSchema`. 
It then transforms the Backblaze data by filtering out rows with null drive models, deriving the manufacturer from the model name, and setting invalid capacity values to null. 
The cleaned DataFrame is stored because its reused across three different collections. 
The main aggregation groups  by drive model to calculate drive-days, failures, average capacity, and annualized failure rate. 
A window function ranks models by annualized failure rate using rank() ordered from highest to lowest AFR. 
The model-level statistics are joined with manufacturer-level statistics to add context, including each model’s share of its manufacturer’s fleet and how its AFR compares with the manufacturer average. 
The final output is written to $BUCKET/backblaze_out/workers_n/failure_rate_by_model.
